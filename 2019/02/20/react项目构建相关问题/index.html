Webpack相关

webpack会假定项目的入口为src/index

webpack.config.js

1. path模块

Node.js path模块提供了一些用于处理文件路径的小工具。

2. path.join() && path.resolve()

path.join()
用于连接路径。该方法的主要作用在于可以正确的使用当前系统的路径分隔符进行连接。【Unix系统:”/“ windows系统:”\”】

path.resolve()
将路径或者路径片段的序列解析为绝对路径。 给定的路径序列从右至左进行处理，直至构造出一个绝对路径。

3. 插件相关

  • HtmlWebpackPlugin

    HtmlWebpackPlugin插件 简化了html文件的创建【不用这个插件我们手动在dist文件夹下放一个html文件，当每次打包完成后，更改html中script引入的js文件名】。该插件对于在文件名中包含每次会随着编译而发生变化的 Hash的webpack bundle是非常有用的。

React相关

1. react && reactDOM

在自定义组件中我们我们使用 class extends React.Component{},在项目的入口文件里使用ReactDOM({<MyApp />, document.getElementById('root')})将项目挂载到id = root 的DOM元素上。

react包含React.createElement``.createClass``.Component``.PropTypes``.Children 这些 API，而 react-dom package 中包含 ReactDOM.render``.unmountComponentAtNode .findDOMNoded等这些API.

1. router4如何像router3那样使用(路由表存在)

在router3中我们使用的是<Router history={history} />,router4中对<Router />元素进行了包装，即我们最常使用的<HashRouter/ >或者<BrowserRouter/ >。在router3中直接使用<Router/ >需要我们手动引入history.
